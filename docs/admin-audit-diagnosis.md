# Admin Audit — Deep Diagnosis

> Generated by trouvable-orchestrator after full inventory, data-flow tracing, and duplication analysis of all 64 admin files, 13 view components, 27 API routes, and 5 data layer modules.

---

## 1. Executive Summary

The Trouvable admin ("Centre de commande") is a well-built operator workspace with premium dark UI, clean data-flow architecture, and correct Supabase wiring. However, it has **one structural gap** and **several design redundancies** that reduce operator efficiency:

| Finding | Severity | Type |
|---------|----------|------|
| No company-level dashboard exists | **High** | Structural gap |
| SEO/GEO scores shown in 3 places | Medium | Redundancy |
| Freshness data duplicated in header + overview | Low | Redundancy |
| Overview aggregates data from 5+ detailed views | Low (intentional) | Design choice |
| "Signaux" is a container, not a real view | Low | IA confusion |
| No dedicated Prompts management in nav group | Low | Missing affordance |

**Key insight:** The data wiring is correct. The overview slice (`getOverviewSlice`) properly aggregates summaries from multiple domains. The problem is not broken architecture — it's **missing a level** (company dashboard) and **minor redundancy** in the client workspace.

## 3. Duplication Audit

### 3.1 SEO/GEO Scores — 3 locations (Medium severity)

| Location | Source | Display |
|----------|--------|---------|
| ClientWorkspaceShell header | `audit` from `useGeoClient()` → base.js → `seoScore`, `geoScore` | Inline text in header |
| GeoOverviewView | `audit` from `useGeoClient()` + `kpis` from overview slice | ScoreRings component |
| GeoAuditView | `audit` from `useGeoClient()` | Full audit analysis with categories |

**Verdict:** Shell header showing scores is redundant with Overview. The Audit view's detailed breakdown is correct (it's the dedicated deep-dive). **Remove scores from shell header** and let Overview serve as the summary, Audit as the detail.

### 3.2 Freshness / Engine Health — 2 locations (Low severity)

| Location | Source | Display |
|----------|--------|---------|
| MissionCommandHeader (shell) | `workspace.latestRunAt` from base.js | Color-coded dot + "Xh ago" |
| EngineHealthStrip (overview) | `kpis` from overview slice | Freshness + parse rate + mention stats |

**Verdict:** Shell header freshness is a quick-glance indicator — acceptable for persistent display. EngineHealthStrip in overview is detailed health. This is **intentional layering**, not harmful. Keep both.

### 3.3 Overview "Mini-Dashboard" Effect (Intentional, Low severity)

The Overview page shows summaries of data that detailed views expand:

| Overview Widget | Detailed View | What Overview Shows | What Detail Shows |
|----------------|---------------|--------------------|--------------------|
| ScoreRings | GeoAuditView | SEO/GEO score numbers | Full category breakdown |
| MissionKpiCards | GeoRunsView | Run count + date | Full run list + diagnostics |
| ActionCenter | GeoAmeliorerView | Top actions by priority | Full queue by status |
| CoverageMeters | GeoCitationsView | Coverage % + visibility | Timeline + host analysis |
| SignalsPanel sources | GeoCitationsView | Top 2 sources | Top 8+ paginated |
| SignalsPanel competitors | GeoCompetitorsView | Top 1 competitor | Full threat analysis |
| RecentActivity | (no dedicated view) | Last 5 activities | — |

**Verdict:** This is correct dashboard design. The Overview is a **command center** showing "what needs attention" across all domains, linking to detail views for full analysis. **No change needed** — this is how operator dashboards should work.

### 3.4 "Signaux" View — Container confusion (Low severity)

`GeoSignalsView` is a tab container that embeds `GeoCitationsView` and `GeoCompetitorsView`. These are also accessible directly via `/citations` and `/competitors` routes (mapped to "signals" in sidebar active state).

**Verdict:** The container pattern works but the nav labeling is slightly confusing. Low priority.

## 4. What's Missing

### 4.1 Company-Level Dashboard (HIGH PRIORITY)

**Current state:** `/admin` redirects to `/admin/clients` which is a portfolio list. There is no aggregate dashboard showing:

- Total portfolio health at a glance
- Cross-client KPI trends (total runs, total actions, average scores)
- Critical alerts across all clients
- Engine utilization metrics
- Operational capacity view

The portfolio page has **some** aggregate KPIs (active mandates, critical/attention/stable counts, health bar, freshness strip), but it's embedded in a paginated client list — not a dedicated command center.

**Impact:** The operator must mentally aggregate across clients. No single "how are we doing overall?" view exists.

### 4.2 Missing Navigation Items

| View | File Exists | In Sidebar | Issue |
|------|------------|------------|-------|
| Prompts (/prompts) | Yes — GeoPromptsView | **No** | Must navigate via URL or Overview QuickNav |
| Continuous (/continuous) | Yes — GeoContinuousView | Conditional* | Only shown when activated |
| Models (/models) | Yes — GeoModelesView | **No** | Must navigate via URL |
| Settings (/settings) | Yes — GeoSettingsView | **No** | Must navigate via URL |

*These are real views with real functionality that operators can't discover through normal navigation.

## 5. What Works Well

1. **Data-flow architecture** — Clean separation: shell (base.js) → overview slice (overview.js) → view slices (per-domain loaders). Each layer has a clear purpose.
2. **Lazy loading** — `useGeoWorkspaceSlice` fetches data on-demand per view. No unnecessary loading.
3. **Portfolio enrichment** — `enrichClientsWithOperationalSignals` adds intelligence (attention levels, reasons) to the client list with proper 21-day windowing.
4. **Error handling** — Consistent error states, loading skeletons, and graceful degradation.
5. **UI quality** — Premium dark theme, Framer Motion animations, responsive layout, consistent `cmd-surface` design language.
6. **Overview as operator hub** — Well-designed action-center pattern with now/next/watch columns. Guardrails system catches real issues.
7. **Sidebar UX** — Contextual navigation that shows client nav only when a client is selected, with smooth AnimatePresence transitions.

### 6.3 Company Dashboard Design

The company dashboard (`/admin/page.jsx`) should show:

1. **Portfolio Health Ring** — Donut chart: critical / needs_attention / watch / stable distribution
2. **KPI Strip** — Active mandates, total open actions, average SEO/GEO score, engine uptime
3. **Critical Alerts Panel** — Cross-client guardrail violations and critical-attention clients
4. **Freshness Overview** — Data freshness across all clients (< 24h / 24-72h / > 72h / none)
5. **Recent Activity Feed** — Last 10 activities across all clients
6. **Quick Access** — Link to portfolio list, new mandate

Data source: New `getCompanyDashboard()` function that aggregates across all clients using existing query patterns from `enrichClientsWithOperationalSignals()` and direct Supabase queries.
